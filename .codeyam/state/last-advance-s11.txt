**Cache-update matrix ‚Äî which refresh-tests run updates the gate cache.**
The audit gate reads the FULL-SUITE `test-cache.json`. Only a full run
updates it: `--changed`, `--test`, `--runner`, and `--partition` update
only the targeted / partition cache(s), and flag-free `refresh-tests` is
gated mid-session (it refuses until `finalize`). So for a brand-new test
file ‚Äî which is untracked and therefore lands in NO partition cache yet ‚Äî
the command that updates the gate cache in one step is
`refresh-tests --force`. Do not cycle through `--partition all` ‚Üí
flag-free ‚Üí `--changed` first; for brand-new (untracked) tests those each
succeed quietly without updating the gate cache. Modifying an
already-tracked test is the `--changed` case (it updates that test's
partition cache).

Read-before-edit discipline (avoids wasted "File has not been read yet" Edit/Write failures):
